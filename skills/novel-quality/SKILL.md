# /novel-quality — 写后质量门控

## 触发

正文写完（`/novel-generate` 完成）后自动运行。也支持独立调用：`/novel-quality 5`。

## 四步质量检查

### Step 1：违禁句硬扫描

```bash
python 变量层/脚本/scan_forbidden_phrases.py 动态层/正文/第{N}章.md
```

读 `稳定层/规则/forbidden_phrases.md` 获取词表。扫描正文，输出命中报告。

**命中必须改。不是润色——是重写句子。**

### Step 2：轮换检查

```bash
python 变量层/脚本/rotate_check.py --chapter {N} --project-root .
```

检查当前章与最近 5 章的：

- 起手骨架是否重复
- 收束方式是否重复
- 核心意象是否重复
- 高频动作是否重复

输出重复项报告。**允许有理由的呼应，不允许机械复现。**

### Step 3：密度回检

章纲事件数是否 ≥ 4 条？若正文实际写出的事件少于章纲 → 标记不完整。

爽点释放检查：本章是否至少兑现了写前确定的 1-2 条爽点？实际释放强度多少？

### Step 4：章记写入

将本章元数据追加到 `动态层/维护/chapter_maintenance.md`：

```markdown
### 第{N}章《标题》
- 章节标识：第{N}章
- 写作时间：{timestamp}
- 本章标签：[战斗/推理/日常/劫后/发展/打脸/爽点/内折/过场]
- 开头切入：[险情/众人观望/租房/...]
- 主叙事距离：[中近景/远景/特写]
- 核心意象：[本章独有意象列表]
- 高频动作：[本章高频动作列表]
- 本章收束方式：[收物定念/收物离场/睡前理局/...]
- 爽点释放情况：强度 X/10
- 下章优先检查项：[待办]
```

只保留最近 5 章详记。更早压缩到"更早记录摘要"。

## 输出

```
📊 质量报告 — 第{N}章《标题》
  违禁句扫描: X处命中 (已修正/待修正)
  轮换检查: Y项与近期重复 (需关注)
  密度回检: Z/10 (≥8通过)
  爽点释放: 强度 W/10
  章记: 已写入
  综合: ✅ 通过 / ⚠️ 有保留项
```
