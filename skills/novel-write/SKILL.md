# /novel-write — 写前流水线

## 触发

用户说"写第N章"时，必须先跑此流水线。全部通过后才允许调用 `/novel-generate`。

## 10 步强制流程

顺序不得颠倒。任一步不通过 → 拒绝写正文 → 输出问题报告。

### Step 1：读随机素材

读 `变量层/随机素材/current_chapter_brief.md`。若缺失，运行：

```bash
python 变量层/脚本/prepare_chapter_brief.py --chapter {N}
```

### Step 2：读维护文档

读 `动态层/维护/chapter_maintenance.md`，重点看：运行态参考、当前提示、最近章节记录、爽点池、点子池。

### Step 3：读作者反馈

读 `动态层/维护/author_feedback.md`。若作者已明确推翻旧写法，旧稿与旧节奏一律降级。

### Step 4：检查手写稿（最高优先级！）

检查 `变量层/作者手写库/` 是否存在当前章对应的手写稿。若存在——**先读手写稿，再读大纲**。手写稿与大纲冲突 → 改大纲服从手写稿。

### Step 5：审大纲后10章

读大纲中"当前章往后10章"。至少抓出 1 个实际问题，重点看：

```
爽点被压 / 收获偏慢 / 阻力不具体 / 关系位移少 / 伏笔效率低 / 章节功能重复
```

### Step 6：组爽点池

从三个来源同时提取：
1. 大纲当前章往后10章
2. 维护文档现有爽点池
3. 本轮随机素材/点子样本

整理出本轮可兑现的 **3-6 条**爽点候选。至少确定 **1-2 条**作为本章正文要打透的爽点。

### Step 7：翻点子池

从维护文档的点子池挑 1-3 个候选，至少采纳 1 个。先改大纲把点子纳入章纲。若点子质量不够：

```bash
python 变量层/脚本/prepare_chapter_brief.py --chapter {N} --refresh
```

### Step 8：判定章节标签

从以下标签中选一个：

```
战斗 / 推理 / 日常 / 劫后 / 发展 / 打脸 / 爽点 / 内折 / 过场
```

标签决定本章的主气压和节奏。

### Step 9：章纲密度闭环

按 `变量层/情节库/density_rubric.md` 拟 **4-6 条有效事件**。目标 **≥ 8/10**。

不够 → 不许拿气氛/停顿/车轱辘对白硬撑。先补情节。还不够 → 重跑脚本补点子：

```bash
python 变量层/脚本/prepare_chapter_brief.py --chapter {N} --refresh
```

可运行校验脚本：

```bash
python 变量层/脚本/validate_chapter_outline.py 动态层/大纲/第{N}章章纲.md
```

不达标不得动笔。

### Step 10：违禁句预判

读 `稳定层/规则/forbidden_phrases.md`。本章不得主动写出高危句核。若预计会频繁碰到这些表达 → 先改章纲和句法，不得靠写后硬扫返工。

## 通过条件

全部 10 步通过后，输出：

```
✅ 写前流水线通过
   章节标签: [标签]
   爽点池: X条候选, Y条本章打透
   点子池: 采纳Z个
   密度评分: X/10
   违禁句预判: 已规避
   → 允许调用 /novel-generate
```

任一步不通过 → 输出具体问题报告，禁止继续。
