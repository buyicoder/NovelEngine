# /novel-feed — 喂书：建立个人风格指纹库

## 目标

将作者的成熟作品（.md / .txt / .epub）喂给 NovelEngine，系统自动拆解为**写作技法单元**，多维度向量化。后续写作时可精确检索"我以前怎么写类似场景"，质量门控可检测风格偏离。

## 核心设计：不只是"把小说分块存起来"

普通的 RAG 是把小说按章切了存向量。那不够。你搜"我以前怎么写战斗场景"，它返回的是整章——你还要自己从 3000 字里找那一段写法。

我们的做法：**把小说拆成"技法单元"——每个单元是一个可复用的写作决策。**

```
一部小说
  → 分章 → 分场景 → 每个场景拆成 5 个维度的向量
      维度1: 内容向量（这个场景发生了什么）
      维度2: 技法向量（怎么写出来的——句式/节奏/转场）
      维度3: 情绪向量（情感曲线——从什么情绪到什么情绪）
      维度4: 对白向量（对话的节奏、密度、风格）
      维度5: 结构向量（这个场景在全书中的位置和功能）
```

搜索"战斗场景 + 短句爆发 + 对白驱动"时，它不会返回整章——它返回**你以前写过的战斗场景中，最符合短句爆发 + 对白驱动的那些具体段落。**

## 流程

### Step 1：导入

```bash
/novel-feed 重生三部曲.epub        # 单部
/novel-feed ./我的作品集/           # 批量
```

支持：`.md` `.txt` `.epub`

### Step 2：多层级分块（不是按章切）

```
第3章《北麓精研》
  ├── 场景1: 租房挂牌（起手骨架：租房起手）
  │   ├── chunk-A: 环境白描 → 类型=场景建立, 技法=远景切中景, 句长=中句
  │   ├── chunk-B: 叶尘挂木牌 → 类型=人物动作, 技法=短句利落, 句长=短句
  │   └── chunk-C: 齐云来看 → 类型=人物登场, 技法=钝答对白, 密度=低
  │
  ├── 场景2: 集市买药（冲突推进）
  │   ├── chunk-D: 世家垄断伏天草 → 类型=阻力建立, 技法=明话+短顶, 爽点=压迫
  │   ├── chunk-E: 沈小晚登场 → 类型=新人物登场, 技法=短顶对白, 情绪=紧张
  │   └── chunk-F: 中毒被救 → 类型=转折, 技法=短句爆发+收物定念, 爽点=释放
  │
  └── 场景3: 睡前理局（收束）
      └── chunk-G: 重新躺回去 → 类型=收束, 技法=睡前理局, 句长=中句, 情绪=下沉
```

每个 chunk 400-800 字，携带 5 个维度的元数据标签。

### Step 3：五维向量化

每个技法单元生成**一个复合向量**：

```
复合向量 = [语义向量 (512维)] + [技法向量 (128维)] + [情绪向量 (64维)]
         = 704 维

语义向量: bge-small-zh-v1.5 → 这个 chunk 在说什么
技法向量: 句长分布(16) + 对白密度(16) + 描写比例(16) + 
          高频词指纹(32) + 节奏标记(16) + 转场类型(16) + 叙事功能(16)
情绪向量: 8 种基本情绪的强度分布 → 映射到 64 维
```

技法向量不依赖模型——它是一组手工特征，直接拼在语义向量后面。保证了"短句爆发"这种风格查询能被精确匹配。

### Step 4：混合搜索

和 Jarvis 相同方案——三层融合：

```
查询: "战斗场景 + 短句爆发 + 对白驱动 + 爽点释放"

  → BM25 关键词: 匹配"战斗""爽点" → 候选池A
  → 语义向量搜索: 匹配内容相似的战斗场景 → 候选池B
  → 技法向量过滤: 筛选句长=短句、对白密度=高 → 候选池C

  → RRF 融合 → Top-15
  → 时间加权（近期作品 ×1.2）→ Top-5
```

**加上元数据预过滤：** 先过滤"类型=战斗场景"+"爽点标记=已释放"，再在过滤后的子集里做向量搜索。不搜全库。

### Step 5：生成风格指纹

```markdown
# 作者风格指纹

## 句长分布
- 短句(≤15字): 42%  |  中句(16-40字): 45%  |  长句(>40字): 13%

## 对白密度: 38%（高于网文平均 25%）→ 对白驱动型

## 描写风格: 白描 78% / 渲染 22%

## 场景类型分布
- 战斗: 15% (短句爆发+爽点释放)
- 日常: 35% (中句缓推+关系推进)
- 推理: 20% (中句细描+信息递进)
- 爽点: 18% (短句爆发+四件套)
- 过场: 12%

## 个人癖好（标记为"允许"，不触发AI味告警）
- "不是……是……": 已出现 47 次 → 允许
- "静了一下": 已出现 23 次 → 允许
- 破折号插入语: 已出现 89 次 → 允许
- 收束句"天一亮，就见分晓": 个人标志性收束 → 允许复用

## 多部作品演化
  早期(重生): 短句35%, 偏华丽
  中期(时层协议): 短句40%, 转白描
  近期(科研修仙): 短句42%, 对白驱动确立
→ 风格正往更简洁、更对白驱动的方向收敛
```

### Step 6：在写作流程中应用

#### novel-write 新增 Style Check

```
写前流水线新增 Step 0:
  → 读取风格指纹
  → 本章标签"战斗" → 检索作者过往战斗场景的 Top-5 写法
  → 本章标签"爽点" → 参考作者自己写过的爽点释放节奏
  → 输出："建议对齐近期风格——短句爆发 + 对白驱动 + 收物定念"
```

#### novel-quality 新增 Style Deviation

```
质量门控新增:
  → 句长分布偏离指纹 ±20% → ⚠️
  → 对白密度突降 → ⚠️
  → 出现指纹中未标记为"允许"的句式 → ❌
  → 技法向量距离 > 0.8（偏离作者的通常写法）→ ⚠️
```

## 为什么这个设计够强

| 对比 | 普通 RAG | NovelEngine 喂书 |
|------|---------|-----------------|
| 分块粒度 | 按章（3000字） | 按技法单元（400-800字） |
| 向量维度 | 1 个语义向量 | 5 个维度复合向量 |
| 搜索方式 | 纯语义 | 元数据过滤 + BM25 + 语义 + 技法过滤 + RRF |
| 风格检索 | 不支持 | 手工特征向量：句长/对白/节奏/转场 |
| 时间衰减 | 无 | 近期作品 ×1.2 |
| 个人癖好 | 可能被误判为 AI 味 | 标记为"允许"，不触发告警 |
