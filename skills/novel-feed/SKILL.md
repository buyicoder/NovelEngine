# /novel-feed — 喂书：建立个人风格指纹库

## 目标

将作者的成熟作品（.md / .txt / .epub）喂给 NovelEngine，系统自动：

1. **拆解结构**：分章 → 提取场景 → 标注叙事功能
2. **提取风格指纹**：句长分布、对话密度、描写比例、高频词汇、节奏曲线
3. **向量化存储**：将每个场景的"写法"嵌入，后续写作时可语义检索
4. **建立风格画像**：生成 `稳定层/风格指纹/author_style.md`

## 流程

### Step 1：导入

```bash
# 单文件
/novel-feed 我的小说.txt

# 整个目录（多部作品）
/novel-feed ./我的作品集/

# epub
/novel-feed 重生三部曲.epub
```

支持格式：`.md` `.txt` `.epub`

### Step 2：结构化

对每部作品自动：

```
分章 → 每章提取：
  ├── 叙事功能标签（战斗/推理/日常/爽点/过场）
  ├── 场景边界（每个场景的开头和结尾）
  ├── 对白/描写/叙述比例
  ├── 句长分布（短句占比/中句/长句）
  ├── 高频词汇 Top 50
  ├── 节奏曲线（每 500 字的情绪强度）
  └── 独特句式（"不是……是……"类的个人癖好——标记为"允许使用"）
```

存入 `变量层/作者风格库/<作品名>/`

### Step 3：向量化

```
每个场景 → bge-small-zh-v1.5 嵌入 → LanceDB
索引维度：
  - 风格向量（句长+节奏+密度）
  - 内容向量（场景类型+情绪+叙事功能）
  - 写法向量（具体句子结构、过渡方式）
```

存入 `变量层/风格向量/`

### Step 4：生成风格指纹

自动生成 `稳定层/风格指纹/author_style.md`：

```markdown
# 作者风格指纹

## 句长偏好
- 短句(≤15字): 42%
- 中句(16-40字): 45%
- 长句(>40字): 13%
→ 偏好干净短促，偶用长句压场

## 对白密度
- 对白占比: 38%（高于网文平均 25%）
→ 对白驱动型叙事

## 描写风格
- 白描: 78%  |  渲染: 22%
→ 重白描，轻华丽

## 高频词
（前 20 个实词）

## 节奏特征
- 开篇：中速推进 × 收尾：利落压场
- 战斗场景：短句爆发 × 日常场景：中句缓推

## 个人癖好（标记为"允许"）
- "不是……是……"：作者惯用，非AI味
- "静了一下"：作者惯用，允许
- 破折号插入语：常见，保留
```

## 后续写作中的应用

### 在 novel-write 中

写前流水线新增一步：**风格对齐检查**

```
Step 0（新增）：读风格指纹
  → 本章标签是"战斗" → 检索作者过往战斗场景的写法
  → 本章标签是"日常" → 检索作者过往日常场景的写法
  → 爽点设计 → 参考作者自己写过的爽点释放节奏
```

### 在 novel-quality 中

质量门控新增一项：**风格偏离检查**

```
若本章句长分布突然偏离指纹 ±20% → ⚠️ 标记
若对白密度突然降到 10%（作者通常 38%）→ ⚠️ 标记
若出现指纹中未标记为"允许"的 AI 高频句式 → ❌ 拒绝
```

## 多部作品的风格演化追踪

如果喂了 3 部以上作品，系统额外生成：

```
风格演化报告：
  早期作品：句长 35%短句，描写偏华丽
  中期作品：句长 42%短句，描写转白描
  近期作品：句长 42%短句，对白驱动确立
→ 作者的风格正在向更简洁、更对白驱动的方向收敛
→ 新章节写作默认对齐近期风格
```
