> 网文创作辅助系统 v2.0 —— 不只是帮你写，是帮你写得**不崩、不水、不AI味**。

NovelEngine 跑在 Claude Code 上。它不替你写书——它替你管设定、控质量、防重复、去 AI 味。你负责创作，它负责不让创作翻车。

## v2.0 有什么新东西

基于一部实际写到第 9 章的科研修仙小说的生产经验，把"怎么让 AI 写出不崩的网文"这件事工程化了。

| 能力 | 说明 |
|------|------|
| 🏭 **三层文件系统** | 稳定层（大纲/设定/规则）+ 变量层（手写稿/随机素材/情节库）+ 动态层（正文/维护/归档）。AI 能改什么、不能改什么，权限边界清楚 |
| ✍️ **写前 10 步流水线** | 爽点池组装 → 点子池翻取 → 密度评分 ≥8/10 → 违禁句预判。全部通过才允许动笔 |
| 💥 **爽点引擎** | 不是"这段写爽一点"——是 6 类爽点候选池 + 四件套释放（压迫→翻盘→众人反应→主角心气）+ 连锁放大 |
| 🔍 **质量门控** | 违禁句硬扫描（20+ 个 AI 高频句式）+ 轮换检查（近 5 章起手/收束/意象/动作不可重复）+ 密度回检 |
| 🚫 **去 AI 味系统** | 写前预判 + 写后硬扫描。命中不是润色——是重写 |
| 📊 **章记 Schema** | 每章 15 个元数据字段。AI 写到第 20 章时不会忘了前 3 章用了什么手法 |

## 三层文件系统

```
项目/
├── 稳定层/                  ← 不轻易改。人类编辑，AI只读
│   ├── 大纲/                 ← 总纲 + 分卷详细大纲
│   ├── 设定集/               ← 世界观/力量体系/人物卡
│   └── 规则/                 ← 系统指令 + 违禁句词表
│
├── 变量层/                  ← 每章刷新。脚本生成，手写稿最高优先级
│   ├── 作者手写库/           ← 🔴 最高优先级！手写稿 > 大纲
│   ├── 随机素材/             ← prepare_chapter_brief.py 生成
│   ├── 情节库/               ← density_rubric.md 密度评分标准
│   └── 脚本/                 ← Python 辅助脚本
│
├── 动态层/                  ← AI运行时写入
│   ├── 正文/                 ← 每章输出
│   ├── 白话剧情/             ← 口语化剧情复述
│   ├── 维护/                 ← chapter_maintenance.md + author_feedback.md
│   └── 归档/                 ← 旧章风格记忆
│
└── 导出/                    ← epub/pdf
```


## 爽点引擎

网文最玄学的"爽"，被量化为三个层级：

```
爽点池（3-6条候选）
  ├── 来源1: 大纲当前章往后10章
  ├── 来源2: 维护文档现有爽点池
  └── 来源3: 随机素材/点子样本

↓ 至少确定 1-2 条本章打透

四件套释放
  压迫 → 翻盘动作 → 众人反应 → 主角心气

↓ 连锁放大
  不只写"一次赢了"，把后续收益、能力外溢、结构性红利全打出来
```

实测效果：第 1 章强度 6/10，第 2 章强度 8/10。


## 去 AI 味系统

```
写前预判
  → 本章不得主动写出 "不是...是..." "像是..." "仿佛..."
  → "这句话里" "听出了" "眼里有" 全部预判规避
  → 预计会高频碰到的，先改章纲和句法

写后硬扫描
  → scan_forbidden_phrases.py 扫描全文
  → 20+ 个 AI 高频句式命中必须改
  → 不是润色——是重写句子

风格许可
  → 允许现代网文词："懵了" "炸了" "上头" "吃瘪" "翻车"
  → 禁止空洞玩梗和满篇论坛黑话
  → 不是禁止现代词，是禁止 AI 式的空洞感
```


## 轮换系统

AI 最大的问题是重复。每章写完后自动检查 4 个维度是否和近 5 章相同：

| 维度 | 示例 | 轮换规则 |
|------|------|---------|
| 起手骨架 | 险情起手 / 众人观望 / 租房起手 | 每章不同 |
| 收束方式 | 收物定念 / 收物离场 / 睡前理局 | 每章不同 |
| 核心意象 | 炸炉+废丹 / 十炉台+五千灵石 / 木牌+破屋 | 每章不同 |
| 高频动作 | 控火数灵石 / 扫地钉木架 / 买药救人 | 每章不同 |

**原则：优先错开，不强求章章全面换新。允许有理由的呼应，不允许机械复现。**

## 为什么不是"AI 一键写小说"

AI 直出小说有三个致命问题：
1. **AI 味诅咒**：读者比你想象中更敏锐。AI 味的小说在签约环节就被毙掉
2. **质量衰减**：前 3 章还行，写到第 20 章 AI 忘了前 3 章用了什么手法——然后开始水
3. **爽点失控**：AI 不会判断"这段该爽了"，它只会平铺直叙

NovelEngine 不是替你写——是确保你（和 AI）在写的每一步，设定不崩、质量不掉、爽点到位、AI 味可控。
